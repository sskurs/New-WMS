WMS Application Field Reference

This document provides a detailed breakdown of all data fields used within the application forms and database schemas.

Required Mandatory Field   AI Powered Field supports AI generation/validation   Read Only System calculated

1. Product Catalog

Defines the items managed by the warehouse. Accessible via Inventory > Catalog > Add/Edit.

General Information

Field NameData TypeDetails & Business Rules
Product Name Required AI String Commercial name. AI validates plausibility on blur. Used for search and display.
SKU Required AI String Stock Keeping Unit. Unique identifier. Can be AI-generated based on Name + Category.
Product Code Required AI String Internal reference code (e.g., PROD-XXXX). Can be AI-generated.
Category Required AI Dropdown (Reference) Links to Category master. AI suggests category based on product name.
Supplier Required Dropdown (Reference) Primary vendor. Links to Supplier master.
Supplier Type Dropdown Options: Local, International. Defaults to Local.
Brand ID Required AI String Brand identifier. Can be AI-generated.
Label ID Required AI String Labeling identifier. Can be AI-generated.
Description Required AI Text Area Detailed description. AI can generate marketing text based on name/category/price.
Product Image File (Base64) Product photo. Max size 2MB.

Pricing & Inventory

Field NameData TypeDetails & Business Rules
Base Price Required Decimal Cost before tax. Must be > 0.
GST Percentage Dropdown (Number) Tax rate. Options: 0, 5, 12, 18, 28.
Final Price Read Only Decimal Calculated as Base Price * (1 + GST/100).
HSN/SAC Code Required String Harmonized System code for taxation.
Pricing Rule Dropdown (Reference) Links to PricingRule master for dynamic discounts.
Unit of Measure Dropdown Stock keeping unit type (e.g., PCS, KG, Box).
Pack Size Number Quantity per unit (e.g., 12 for a Dozen). Default 1.
Reorder Point Required Number Minimum stock threshold triggering low stock alerts.

Packaging & Variants

Field NameData TypeDetails & Business Rules
Packaging Type Required Dropdown e.g., Primary, Secondary, Tertiary.
Materials Used Checkbox List e.g., Cardboard, Plastic, Foam.
Handling Dropdown Storage environment (Standard, Refrigerated, etc.). Used for AI location suggestions.
Shelf Life Required Number (Days) Days until expiration.
Variants Key-Value List Custom attributes (e.g., Color: Red, Size: XL).

2. Stock Adjustments

Used to manually correct inventory levels. Accessible via Inventory > Adjustments > New.
Field NameData TypeDetails & Business Rules
Mode Radio Quantity (adjust by unit count) or Value (adjust by total monetary value).
Product Required Dropdown (Reference) Item to adjust.
Location Required Dropdown (Reference) Physical location of stock (Bin/Shelf or Receiving Dock).
Adjustment Type Required Dropdown Increase (Found stock) or Decrease (Damage/Loss/Theft).
Quantity Required Number Amount to adjust. Must be > 0. If decreasing, cannot exceed current stock.
Adjustment Value Number If Mode is 'Value', input total currency amount. System calculates equivalent unit quantity.
Reason Dropdown Audit reason (e.g., "Damaged goods", "Stocktaking results").

3. Customer Orders

Sales orders to be fulfilled. Accessible via Orders > Processing > New.

Header Details

Field NameData TypeDetails & Business Rules
Customer Name Required AI String AI validated for name plausibility.
Email Address String Format validation required (email@domain.com).
Phone Number String Must be exactly 10 digits (numeric).
Priority Dropdown High, Medium, Low. Affects sorting in workflow.
Notes Text Area Internal instructions or customer remarks.

Shipping Address

Field NameData TypeDetails & Business Rules
Address Line 1 Required String Street address/House number.
Address Line 2 Required String Apartment, Suite, Unit, etc.
State Required Dropdown List of Indian States.
City Required Dropdown Filtered based on selected State.
Zip Code Required String Must be 6 digits.

Order Items

Field NameData TypeDetails & Business Rules
Product Required Dropdown (Reference) Selected from catalog. Displays available stock.
Quantity Required Number Must be > 0. Cannot exceed Available Stock.
Unit Price Read Only Decimal Auto-filled from Product. Applies PricingRules if applicable.

4. Returns (RMA)

Return Merchandise Authorization. Accessible via Orders > Returns > New.
Field NameData TypeDetails & Business Rules
Order ID Search / Select Must select an existing Order with status Shipped or Completed.
Items to Return Required Selection List User selects specific items and quantity from the original order. Max qty = Original qty - Already returned.
Reason Required Dropdown e.g., "Defective Product", "Wrong Item".
Resolution Required Dropdown e.g., "Full Refund", "Exchange".
Contact Phone String 10-digit validation. Pre-filled from original order.
Photo File (Base64) Evidence of damage/condition.
Detailed Description Text Area Additional context for the return.

5. Supplier Database

Vendors for inbound stock. Accessible via Supplier Order > Suppliers > Add.
Field NameData TypeDetails & Business Rules
Supplier Name Required AI String Company name. AI validated.
Contact Person Required AI String Name of representative. AI validated.
Phone Required String 10-digit validation.
Email Required String Email format validation.
Address Fields String / Dropdown Country (India), State, City, Zip (6 digits), Address Line.
Status Dropdown Active or Inactive. Inactive suppliers are hidden in PO creation.
Rating Slider (1-5) Internal vendor rating.
Website String Optional URL.

6. Purchase Orders

Orders sent to suppliers. Accessible via Supplier Order > Purchase Orders > Create.
Field NameData TypeDetails & Business Rules
Supplier Required Dropdown (Reference) Filters product list to items linked to this supplier.
Product Required Dropdown (Reference) Items offered by selected supplier.
Quantity Required Number Must be > 0.
Cost/Unit Decimal Auto-filled from supplier price list, editable.
Status Read Only String Lifecycle: Draft -> Issued -> Partially Received -> Received.

7. Location Management

Physical storage spots. Accessible via Locations > Add Location.
Field NameData TypeDetails & Business Rules
Warehouse Required Dropdown Parent facility.
Zone Required Dropdown Logical area (e.g., Zone A).
Code Required AI String Unique ID (e.g., A1-R1-S2). Can be AI-generated based on warehouse/zone.
Name Required AI String Descriptive name. Can be AI-generated from Code.
Type Dropdown Aisle, Rack, Shelf, Bin.
Capacity Number Max units allowed. Required for bins/shelves.
Status Dropdown Available, Occupied, Reserved, Maintenance.
Address Text Area Used for geocoding.
Lat/Long AI Number Geographic coordinates. AI fetches from Address.

8. Warehouse Configuration

High-level facility settings. Accessible via Configure > Warehouse > New.
Field NameData TypeDetails & Business Rules
Name Required String Facility identifier.
Type Required Dropdown e.g., Distribution Center, Fulfillment Center.
Phone Required String 10-digit validation.
Address Fields String / Dropdown Line 1, Line 2, State, City, PIN (6 digits).

9. User Management

System access control. Accessible via Users > Add User.
Field NameData TypeDetails & Business Rules
Username Required String Login ID.
Role Required Dropdown Admin, Warehouse Manager, Picker, Receiver, Analyst.
Password Password Required on creation. Validated for match with Confirm Password.
Full Name Required AI String AI validated.
Official Email Required String Format validation.
Personal Email String Format validation.
Date of Joining Required Date Cannot be in the future.
Contact No Required String 10-digit validation.
Salutation Dropdown Mr., Mrs., Ms., Dr.
Gender Dropdown Male, Female, Other.
Remarks Text Area Internal notes.

10. Pricing Rules

Dynamic pricing logic. Accessible via Configure > Master Data > Pricing Rules > Add.
Field NameData TypeDetails & Business Rules
Rule Name Required String Identifier (e.g., "Bulk Discount").
Description Text Area Optional description.
Priority Dropdown High, Medium, Low. Determines execution order.
Quantity Limits Number Min Qty / Max Qty conditions for rule to apply.
Date Range Date Start and End dates for validity. Start date cannot be past for new rules.
Action Type Dropdown Percentage Discount, Fixed Price, Percentage Markup.
Action Value Number The discount %, fixed amount, or markup %. Must be > 0.
Status Toggle Active / Inactive.
