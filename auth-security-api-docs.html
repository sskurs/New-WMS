Authentication & Security API Docs

This document details the backend requirements for the password management features in the WMS application.

1. Change Password

Allows an authenticated user to change their own password. This action usually requires the user to verify their identity by providing their old password.

Endpoint Details

POST /api/authentication/Auth/ChangePassword

Request Payload

The client will send the user's ID along with the current (old) password for verification and the new password.

{
    "userId": 10, // The ID of the currently logged-in user
    "oldPassword": "CurrentPass123!",
    "newPassword": "NewStrongPass456$"
}

Success Response (200 OK)

{
    "status": "success",
    "message": "Password changed successfully."
}

Error Response (400 Bad Request)

{
    "status": "failure",
    "message": "Incorrect old password."
}

2. Forgot Password Flow

This is a two-step process for unauthenticated users who cannot access their account.

Step 1: Request OTP

POST /api/authentication/Auth/ForgotPassword

The user provides their registered email address. The backend should generate a temporary OTP (One Time Password), save it (hashed) in the database with an expiration time, and email it to the user.

Request Payload

{
    "emailId": "user@example.com"
}

Success Response (200 OK)

{
    "status": "success",
    "message": "OTP sent to your email address."
}

Error Response (404 Not Found)

If the email does not exist in the system.

{
    "status": "failure",
    "message": "Email address not found."
}

Step 2: Reset Password

POST /api/authentication/Auth/ResetPassword

The user submits the email, the OTP they received, and the desired new password. The backend must verify the OTP is valid and not expired before updating the password.

Request Payload

{
    "emailId": "user@example.com",
    "otp": "482915",
    "newPassword": "BrandNewPass789!"
}

Success Response (200 OK)

{
    "status": "success",
    "message": "Password reset successfully."
}

Error Response (400 Bad Request)

{
    "status": "failure",
    "message": "Invalid or expired OTP."
}
