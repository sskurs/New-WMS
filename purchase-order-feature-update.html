Feature Update: Dynamic Product Loading by Supplier

This document outlines a functional enhancement to the Purchase Order (PO) creation form, introducing a backend-driven approach for populating product dropdowns based on the selected supplier.

Change Request

The core of this update is to make the product selection process more intuitive and error-proof by dynamically fetching and displaying only the products associated with the chosen supplier.

New Behavior: When a user selects a supplier, an API call will be made to the backend to retrieve a list of products exclusively offered by that supplier. The product dropdowns on the form will then be populated with this filtered list.

User Workflow

  1. Initial State: The user navigates to the "Create Purchase Order" form. The product dropdowns are disabled.
  2. Supplier Selection: The user selects a supplier from the main "Supplier" dropdown menu.
  3. API Call: The frontend makes an API request to a new endpoint, passing the selected supplier's ID.
  4. Product Population: The product dropdowns are enabled and populated with the list of products returned from the API.
  5. Supplier Change: If the user changes the supplier, the process repeats. Any previously selected products on line items are cleared to ensure data integrity.

API Specification

A new backend endpoint is required to support this feature.

Get Products by Supplier ID

GET /api/Product/GetBySupplierId?supplierId={id}
  • Description: Fetches a list of products associated with a specific supplier. This is designed to be a lightweight endpoint for populating dropdowns efficiently.
  • URL Parameters:
    • supplierId (required): The unique identifier (integer) of the supplier.

Expected Success Response (200 OK)

The endpoint should return a JSON array of product objects. Each object should contain the essential information needed for the dropdown (ID and Name) and any other useful data like SKU.

[
  {
    "productId": 101,
    "productName": "Quantum SSD 512GB",
    "sku": "ELEC-QUA-512",
    "cost": 4500.00 // The supplier-specific cost price
  },
  {
    "productId": 102,
    "productName": "Quantum SSD 1TB",
    "sku": "ELEC-QUA-1024",
    "cost": 8200.00
  },
  {
    "productId": 125,
    "productName": "Advanced Cooling Fan",
    "sku": "ELEC-FAN-ADV",
    "cost": 750.50
  }
]

Frontend Implementation Notes

  • The components/suppliers/PurchaseOrderForm.tsx will be modified.
  • An useEffect hook should be implemented to watch for changes to the selected supplierId.
  • When supplierId changes, a new API call should be made to the endpoint defined above.
  • A new state variable (e.g., supplierProducts) will be needed to store the fetched list of products.
  • Loading and error states for the API call must be handled to provide user feedback (e.g., showing a spinner in the product dropdown while loading).

Benefits of this Update

  • Reduced Errors: Prevents users from accidentally ordering a product from the wrong supplier.
  • Improved Performance: Avoids loading the entire product catalog on the PO page, leading to faster load times.
  • Enhanced User Experience: Creates a more logical and efficient workflow for the user.
  • Scalability: The system will perform well even as the number of products grows into the thousands.
