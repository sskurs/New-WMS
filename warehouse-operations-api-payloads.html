Warehouse Operations API Payloads

This document details the JSON payloads for key warehouse operations like Picking, Packing & Shipping, and Cycle Counting.

Picking Operations

Picking involves gathering items from their storage locations to fulfill a customer order. The process starts by creating a pick list for an order that is in the 'Processing' state.

Create Pick List

POST /Order/update

To create a pick list, you update the parent order's status to 'Ready for Pickup' and include a `pickLists` array with the items to be picked. The system generates a pick list based on the order items and their stock locations.

Request Payload

{
    "orderManagement": {
        "orders": [
            {
                "id": 205,
                "status": "Ready for Pickup",
                // ... other order fields
            }
        ],
        "pickLists": [
            {
                "id": 0, // 0 for creation
                "orderId": 205,
                "status": "Pending",
                "items": [
                    {
                        "productId": 101,
                        "quantity": 2,
                        "locationId": 56, // ID of the bin/shelf
                        "picked": false
                    },
                    {
                        "productId": 102,
                        "quantity": 1,
                        "locationId": 58,
                        "picked": false
                    }
                ]
            }
        ],
        "packedOrders": [],
        "rmAs": []
    }
}

Update Pick Status

PUT /operations/picklists/{pickListId}/items/{productId}

As a picker scans or manually confirms an item has been picked, this endpoint is called to update the status of that specific item on the pick list.

Request Payload

{
    "picked": true
}

Packing & Shipping Operations

Once all items on a pick list have been marked as 'Picked', the order moves to the packing and shipping stage.

Pack Order

POST /operations/picklists/{pickListId}/pack

This endpoint is called when a packer confirms that all items for an order are packed and ready. It creates a `PackedOrder` record and updates the original order's status.

Request Payload

{
    // Empty payload, the pickListId in the URL is sufficient.
}

Ship Order

POST /operations/packedorders/{packedOrderId}/ship

This final step marks the packed order as shipped. It updates the `PackedOrder` and the original `Order` statuses to 'Shipped' and records the user who performed the action.

Request Payload

{
    "userId": "1" // ID of the user shipping the order
}

Cycle Counting Operations

Cycle counting is an inventory auditing procedure where a small subset of inventory is counted on a given day. This ensures inventory accuracy without shutting down the warehouse for a full count.

Create Cycle Count Task

POST /operations/cyclecounts

Initiates a new cycle count task. The payload contains a list of items (product and location pairs) that need to be counted.

Request Payload

{
    "itemsToCount": [
        {
            "productId": "p1",
            "locationId": "loc-101"
        },
        {
            "productId": "p5",
            "locationId": "loc-103"
        }
    ]
}

Update Counted Quantity

PUT /operations/cyclecounts/{cycleCountId}/items/{itemId}

Submits the actual counted quantity for a specific item within a cycle count task. This is done for each item in the task list.

Request Payload

{
    "countedQuantity": 98 // The actual number of items counted
}

Finalize Cycle Count

POST /operations/cyclecounts/{cycleCountId}/finalize

Once all items in a task have been counted, this endpoint is called to finalize the count. The system will compare the counted quantities with the system quantities and create stock adjustments for any discrepancies.

Request Payload

{
    "userId": "1" // ID of the user finalizing the count
}
