Implementation Guide: RFID/QR Code Scanning

This document outlines the technical approach for integrating RFID and QR code scanning capabilities into the WMSPro application. The goal is to accelerate workflows such as receiving, put-away, picking, and cycle counting by reducing manual data entry and minimizing errors.

High-Level Workflow

  1. Initiate Scan: A user clicks a "Scan" button or focuses an input field on a relevant page (e.g., Cycle Counting).
  2. Capture Data:
    • QR Code: The device's camera is activated to scan a QR code.
    • RFID: A connected hardware scanner reads an RFID tag and inputs the data (typically via keyboard emulation).
  3. Data Lookup: The captured data (e.g., a product SKU or a location code) is sent to the backend via a dedicated API endpoint.
  4. Backend Processing: The backend queries the database to find the entity (Product, Location, etc.) matching the scanned code.
  5. Frontend Action: The backend returns the entity's data. The frontend uses this data to automatically select the item, pre-fill a form, or display relevant details, completing the action.

Database Design

No major schema changes are required. The implementation will leverage existing unique identifiers on the core tables. It is critical, however, to ensure these columns are indexed for fast lookups.

Key Columns for Encoding

  • Products Table: The SKU column should be used as the unique identifier encoded in QR codes for products.
  • Locations Table: The LocationCode column should be used for QR codes on bins, shelves, and racks.

Indexing Recommendation

-- Ensure unique identifiers are indexed for fast lookups
CREATE UNIQUE INDEX idx_products_sku ON Products(SKU);

CREATE UNIQUE INDEX idx_locations_code ON Locations(LocationCode);

Backend Implementation

A single, versatile endpoint is needed to handle lookups for any scanned code.

Lookup Endpoint

GET /api/lookup/{code}

This endpoint will take a scanned code and attempt to find a matching Product, Location, or other entity in the database.

Behavior

  1. The endpoint receives the code from the URL.
  2. It first queries the Products table for a matching SKU.
  3. If a product is found, it returns a JSON object with a type: 'product' and the product data.
  4. If not found, it queries the Locations table for a matching LocationCode.
  5. If a location is found, it returns a JSON object with a type: 'location' and the location data.
  6. If no match is found in any relevant table, it returns a 404 Not Found error.

Success Response Example (Product Found)

// STATUS 200 OK
{
  "type": "product",
  "data": {
    "productId": 101,
    "productName": "Quantum SSD 512GB",
    "sku": "ELEC-QUA-512",
    "price": 4500.00,
    // ... other product fields
  }
}

Frontend Implementation

The frontend will handle both camera-based QR scanning and keyboard-wedge RFID input.

1. QR Code Scanning (Camera)

This requires requesting camera permissions from the user. We must add "camera" to the requestFramePermissions array in metadata.json.

Note: The application will need a library like jsqr to decode the video stream from the camera in real-time.

Example `QrScanner` Component

// components/inventory/QrScanner.tsx

import jsQR from 'jsqr';

  onScan: (data: string) => void;
  onClose: () => void;
}

const QrScanner: React.FC<QrScannerProps> = ({ onScan, onClose }) => {
  const videoRef = useRef<HTMLVideoElement>(null);
  const canvasRef = useRef<HTMLCanvasElement>(null);

  useEffect(() => {
    // ... code to access camera with navigator.mediaDevices.getUserMedia ...
    // ... code to draw video to canvas and scan for QR code with jsQR ...
    
    const tick = () => {
      // ... scanning logic ...
      const code = jsQR(imageData, width, height);
      if (code) {
        onScan(code.data);
        onClose();
      } else {
        requestAnimationFrame(tick);
      }
    };
    requestAnimationFrame(tick);
  }, [onScan, onClose]);

  return (
    <div className="fixed inset-0 bg-black/80 z-50">
      <video ref={videoRef} style={{ width: '100%', height: '100%' }} />
      <canvas ref={canvasRef} className="hidden" />
      <button onClick={onClose}>Close</button>
    </div>
  );
};

2. RFID Scanning (Keyboard Wedge)

Most handheld RFID scanners can be configured in "keyboard wedge" mode, where they emulate a keyboard and "type" the scanned data, followed by an "Enter" key press. We can capture this with a simple input field.

Example `RfidInput` Component

// components/inventory/RfidInput.tsx
import React, { useState, useEffect } from 'react';

interface RfidInputProps {
  onScan: (data: string) => void;
}

const RfidInput: React.FC<RfidInputProps> = ({ onScan }) => {
    const [value, setValue] = useState('');

    const handleKeyDown = (e: React.KeyboardEvent<HTMLInputElement>) => {
        if (e.key === 'Enter' && value.trim()) {
            onScan(value.trim());
            setValue('');
        }
    };

    return (
        <input
            type="text"
            placeholder="Scan RFID tag..."
            value={value}
            onChange={e => setValue(e.target.value)}
            onKeyDown={handleKeyDown}
            autoFocus
        />
    );
};

Integration Examples

Cycle Counting

On the Cycle Counting page, a "Scan Item" button would open the QrScanner modal. Upon scanning a product's QR code, the frontend would call /api/lookup/{sku}. If a product is returned, the form would automatically find that product in the count list and focus the "Counted Quantity" input for the user.

Put-Away

During the put-away process, the user first selects an item to move. Then, they could click "Scan Location". Scanning a bin's QR code would call /api/lookup/{locationCode}. On success, the system would confirm that this is a valid location and automatically pre-fill it as the destination, ready for the user to confirm the put-away action.
