Purchase Order API Payloads

This document provides the clear JSON payload structures required for creating, updating, and retrieving Purchase Orders (POs) in the WMSPro system, based on the specified database schema.

Create Purchase Order

POST /api/warehouse-management/purchase-order/create

To create a new purchase order, send a POST request with the following JSON structure. Note that `purchaseOrderId` and item `id` fields should be set to `0`, as they will be generated by the database.

Request Payload Example

{
  "purchaseOrderId": 0, // Use 0 for creation
  "supplierId": 2,
  "fkWarehouseId": 7,
  "fkLocationId": 7,
  "orderNumber": "PO-2025-1001", // Can be generated by frontend or backend
  "orderDate": "2025-10-04T01:59:24.341Z", // ISO 8601 format
  "expectedDeliveryDate": "2025-10-20T00:00:00.000Z",
  "totalAmount": 1000000.00, // Calculated sum of (quantity * costPerUnit)
  "status": "Pending",
  "remarks": "Urgent order for Q4 sale.",
  "createdBy": 1, // User ID of the creator
  "purchaseItems": [
    {
      "id": 0, // Use 0 for creation
      "productId": 1,
      "quantity": 100,
      "costPerUnit": 2500.00,
      "amount": 250000.00, // quantity * costPerUnit
      "createdBy": 1,
      "receivedQuantity": 0 // Always 0 on creation
    },
    {
      "id": 0,
      "productId": 2,
      "quantity": 5000,
      "costPerUnit": 150.00,
      "amount": 750000.00,
      "createdBy": 1,
      "receivedQuantity": 0
    }
  ]
}

Update Purchase Order

PUT /api/warehouse-management/purchase-order/update

To update an existing purchase order, send a PUT request with a similar structure. You must provide the existing `purchaseOrderId` and the `id` for each `purchaseItem` that is being updated. New items can be included with `id: 0`.

Request Payload Example

{
  "purchaseOrderId": 305, // The ID of the PO to update
  "supplierId": 15,
  "fkWarehouseId": 5,
  "fkLocationId": 8,
  "orderNumber": "PO-2025-0001",
  "orderDate": "2025-10-04T02:03:52.518Z",
  "totalAmount": 465000.00,
  "status": "Approved",
  "modifiedBy": 1, // User ID of the modifier
  "purchaseItems": [
    {
      "id": 408, // The existing ID of this line item
      "productId": 3,
      "quantity": 2000,
      "costPerUnit": 120.00,
      "amount": 240000.00,
      "modifiedBy": "9", // User ID of the modifier
      "receivedQuantity": 500 // Updated received quantity
    },
    {
      "id": 0, // A new item added during an update
      "productId": 4,
      "quantity": 75,
      "costPerUnit": 3000.00,
      "amount": 225000.00,
      "modifiedBy": "9",
      "receivedQuantity": 0
    }
  ]
}

Get All Purchase Orders

GET /api/warehouse-management/purchase-order/get-all

Retrieves a list of all purchase orders. The response will be an array of purchase order objects.

Success Response Payload Example

[
  {
    "purchaseOrderId": 305,
    "supplierId": 15,
    "orderNumber": "PO-2025-0001",
    "orderDate": "2025-10-04T02:03:52.518Z",
    "totalAmount": 465000.00,
    "status": "Approved",
    // ... other purchaseorder fields ...
    "items": [
      {
        "id": 408,
        "productId": 3,
        "quantity": 2000,
        "costPerUnit": 120.00,
        "receivedQuantity": 500,
        // ... other purchaseitem fields ...
      }
    ]
  },
  // ... more purchase order objects
]

Get Purchase Order by ID

GET /api/warehouse-management/purchase-order/get-by-id?id={id}

Retrieves a single purchase order by its unique ID.

Success Response Payload Example

{
  "purchaseOrderId": 305,
  "supplierId": 15,
  "fkWarehouseId": 5,
  "fkLocationId": 8,
  "orderNumber": "PO-2025-0001",
  "orderDate": "2025-10-04T02:03:52.518Z",
  "totalAmount": 465000.00,
  "status": "Approved",
  "modifiedBy": 1,
  "items": [
    {
      "id": 408,
      "productId": 3,
      "fkPurchaseOrderId": 305,
      "quantity": 2000,
      "costPerUnit": 120.00,
      "amount": 240000.00,
      "modifiedBy": "9",
      "receivedQuantity": 500
    },
    {
      "id": 409,
      "productId": 4,
      "fkPurchaseOrderId": 305,
      "quantity": 75,
      "costPerUnit": 3000.00,
      "amount": 225000.00,

      "receivedQuantity": 1000
    }
  ]
}
